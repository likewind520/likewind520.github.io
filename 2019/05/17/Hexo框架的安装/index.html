Hexo框架的安装

Hexo 博客的诞生分三步

第一步 环境的搭建

需要node.js和git Bush的支持,那么需要安装node.js和git Bush;在GitHub上首次登录注册后需要注册ssh-key,怎么得到ssh-key?
首先先检测ssh-key是否存在?在桌面打开gitbush,输入$ ls~/.ssh/id_rsa.pub如果出现文件路径,则存在,否则生成ssh-key.
其次生成ssh-key,输入$ ssh-keygen -t rsa -C “your@email.com“,注意第一次使用GitHub需要先设置自己的用户和邮箱.如:
$ git config -global user.name “yourname”
$ git config -global user.email “your@email.com“
再次打开你的id_rsa.pub,输入cat ~/.ssh/id_rsa.pub
拷贝内容到GitHub上的ssh-key就可以了.
最后测试:$ ssh -T git@github.com ##### 第一次期间若出现是否continue,输入yes出现hi…(加上你的用户名),说明成功了.

第二步 安装Hexo框架

由于在国外下载慢,改为中国的淘宝镜像.在任何地方用gitbush输入npm install -g cnpm –registry=https://registry.npm.taobao.org
然后在创建好的blogs中输入cnpm install hexo-cli -g 即可下载hexo
最后初始化hexo, hexo init 完成.
输入hexo clean 清除public 然后输入hexo g(生成public) 最后输入hexo s(开启服务器)
在地址栏中输入localhost:4000,出现默认的博客页面即为成功.

第三步 把写好的博客推送到GitHub上去.

首先需要在GitHub上建立一个仓库.加号”+”New repository =>Repository name与Owner必须一致,如”likewind520/likewind520.github.io” 不然各种问题.
其次在settings的最下面GitHub Pages修改成主分支master.页面刷新后出现新的地址https://likewind520.github.io 即为你以后访问的地址.

最后输入hexo d (部署)即推送到GitHub上.
每次内容更改都需要执行hexo clean =>hexo g =>hexo s测试完成后 hexo d 推送
几个常用命令:hexo new post “我的第一篇文章” 和hexo n “我的第一篇文章”效果一样.都会默认在source文件下的_posts文件下 生成”.md”的文件.
hexo n page tags 在source下生成一个类似于_posts的tags文件夹.
